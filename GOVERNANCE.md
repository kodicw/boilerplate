# Governance

This document outlines the decision-making process for this repository.

## Goals

1. **Stability**: Template changes should not break existing projects.
2. **Reproducibility**: All templates produce deterministic, reproducible environments.
3. **Transparency**: Make it clear how and why decisions are made.

## Roles

### Maintainers

Maintainers have write access and are responsible for reviewing and merging pull requests. They ensure contributions align with the project's [Contributing Guide](./CONTRIBUTING.md).

### Contributors

Anyone who submits a pull request or opens an issue. Contributors must follow the [Code of Conduct](./CODE_OF_CONDUCT.md).

## Decision Process

### Template Changes

Minor updates (dependency bumps, bug fixes) follow the standard [Pull Request Process](./CONTRIBUTING.md#pull-request-process).

Major changes (new templates, structural overhauls) follow:
1. **Draft**: A contributor opens an issue or draft PR with a proposal.
2. **Review**: Maintainers and community provide feedback.
3. **Approval**: At least one maintainer must approve.
4. **Implementation**: Once approved, the change is implemented and documented.

## Conflict Resolution

In the event of a disagreement, the final decision rests with the lead maintainer. We strive for consensus, but progress should not be blocked indefinitely.
